API Reference Page

Welcome to the AFE news page and API. This page is aimed at showing the different news that appear in the world. Thus, this platform is focused on serving the different news to beign consumed by your journal.

How to use the API

In order to consume some information from the API, we need to get a token by registering in the application. The token enable us to have fully access to the information stored. Indeed, the following table show us the operations and desirable results that our API offers:

URL Operation Acess Description
/users PUT Open Register a user in the application. As a result, we obtain a user inforamtion including the token
/users/login POST Open Login a user into the API (check if a user introduce a correct login)

Detail API Operations

This part of the documentation is focused on showing the inputs and outputs of each API operation.

Register a User

For registering a user, we need to pass for the API a user with the username, email and password:

      
        "user":
          "email":"rick@afenews.io", 
          "password":"rick", 
          "username":"rick"

As a result, we obtain the complete user informatio including the token for being used in the application: 

      
          "user": 
            "username": "rick",
            "email": "rick@afenews.io",
            "token": "SPECIFIC TOKEN HERE"

Login user

In order to login a user into the application, we need to pass a user and password: 

    
        "user":
          "email":"rick@afenews.io", 
          "password":"rick"

As a response of the call, the application shows the same user information as the register if everithing goes fine 

      
          "user": 
            "username": "rick",
            "email": "rick@afenews.io",
            "token": "SPECIFIC TOKEN HERE"
